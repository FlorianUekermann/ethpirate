Manual

This manual consists of two parts:
  1. A description of the pirate game
  2. An example game with user interface screenshots, that describes all user interaction in detail.

1. The pirate game

In the pirate game the player is part of a band of pirates. The group consists of 5 pirates, which are organized in a strict hierachy:
  1. Captain
  2. First mate
  3. Pirate 1
  4. Pirate 2
  5. Pirate 3

The band of pirates found a treasure of 100 coins. The captain decides how many of the coins he wants to share with each individual crew member. After deciding on a split, the crew decides, by open ballot, whether they accept this split proposal or not.
If half or more pirates vote for accepting the proposal, the game ends and each pirate receives the respective number of coins. If more than half of the pirates vote for rejecting the proposal, the captain is killed. With the position of captain vacant, the first mate becomes the new captian, Pirate 1 becomes the new first mate, etc. The remaining pirates repeat this process until a proposal is accepted.
For every action, the mist browser will ask the player to confirm the resulting transaction and display the corresponding data and transaction fees. Aside from the transaction fee, no value is transferred.

Note: The captain always votes to accept the proposal. This ensures that the game ends after at most 3 rejected proposals.

2. Example game and user interface

The following screenshots show a whole game from the perspective of one player, who encounters all possible situations that require a player action. Due to the nature of Ethereum, an action will typically take between 10 seconds and a minute to be processed. During this time the user interface will not reflect the action taken by the player.

Note: Very rarely a transaction can get lost. If you believe this happened, click the button corresponding to your action again. There is no danger in sending redundant actions, since only the first action it received from a player during every step of the game will be processed.

2.1 Interface overview and joining

When the player opens the user interface for the first time, the interface resembles picture below. The user interface consists of three parts:
Top (table)
Summarizes all important information about the current situation.
Middle (boxes)
Shows text boxes and buttons, that inform the user about possible actions.
Bottom (history)
Shows the history of the game as a list of bullet points. The most recent event is listed on top.
Click the join button to participate in the game. The first player to join will be the captian, the second will be first mate etc. After the joining, the player's table column will contain the word 'You' and the text boxes below the table will inform the player about possible actions. The player does not have to wait for other players to join before taking actions.

2.2 Waiting for a proposal

In this example the player was the second participant to join. Therefore there is no possible action until the captain has proposed a split.

2.3 Voting on a proposal

Once the captain has proposed a split, the number of coins, that the captain wants to give to each player is shown in the table (see "Split"-row). The player can now vote to accept or reject the proposal by clicking the respective button in the text box below the table. Once the game has processed a players vote, a green checkmark (accept) or red cross (reject) appears in the corresponding column of the table "Votes"-row. In this example the player voted to reject the proposal.

2.4 Proposing a split

In this example, the previous captains split proposal was rejected. The player is the new captain (note the changed avatars and ranks). To choose a split, the player edits the input fields in the table, that are highlighted by a green outline. After editing the split proposal, the player clicks on the "Propose split" button in the text box below the table. When proposing a split, the captain automatically votes for acceptance and can not take any further action until the vote ends.

2.5 Payment

Once a split proposal was accepted, the player needs to click on the "Get reward" button to receive the ETH corresponding the player's share of the 100 coins .
